# Daat Locus 0.2.0

## Highlights

- Added an embedded WebUI for operating Daat Locus from the browser, with authenticated access, session navigation, agent chat, status views, logs/settings pages, and live activity rendering.
- Added Manager-routed multi-session support. General sessions and project-scoped code sessions can coexist, with generated titles and quick navigation.
- Added the Coding app powered by `scope-engine`, enabling project-aware search, read, edit, and propagation review across multiple languages.
- Added desktop tray support so the daemon can run more naturally in the background on desktop platforms.
- Improved model and provider setup with dynamic model catalog lookup, Codex OAuth import, Ollama support, OpenAI-compatible Responses API support, reasoning options, and vision-capability handling.

## WebUI

- Added an embedded browser UI that is included in release binaries.
- Added authenticated navigation with Agent, Status, Logs, and Settings surfaces.
- Added agent chat with live activity history, message expansion, activity folding, code highlighting, file-edit rendering, and image attachments when the configured model supports vision.
- Added a session sidebar with generated session titles, active-session navigation, and quick switching between sessions.
- Added pending-input controls for queued user messages.
- Added runtime interrupt controls from the WebUI.
- Added status cards for runtime state, current plan, model/context usage, token usage, Telegram approval status, and optimization activity.

## TUI

- Added pending-input queue controls.
- Added text selection support, including grouped selection behavior.
- Added command history seeded from existing sessions.
- Improved slash-command UX with panels, better completions, and quieter successful toggle flows.
- Improved activity rendering for terminal output, file edits, thinking cells, runtime status, and folded history.

## Sessions and Runtime

- Added Manager-routed multi-session architecture: clients talk to the Manager daemon, while each session owns isolated runtime state.
- Added project-scoped sessions through `daat-locus code <project-dir>`.
- Added generated session titles and session quick navigation.
- Improved session lifecycle handling for shutdown, deletion, restart, and unresponsive daemon port reuse.
- Added queued user input handling with visibility and controls in WebUI/TUI.
- Hardened runtime turn interruption, compaction recovery, event completion, browser backend lifetime, terminal output capture, and app-state handling.

## Coding and Project Tools

- Added the Coding app for project-aware code operations.
- Added `scope-engine` integration for content search, hash-anchored code reads, hash-anchored edits, and propagation review.
- Added multi-language parsing/LSP coverage for Rust, Python, TypeScript/JavaScript, Go, Java, C/C++, Ruby, and PHP.
- Added structured file-edit tools for non-source files and structured code-edit tools for source files.
- Improved project-scoped coding sessions so terminal and Coding app work from the target project directory.

## Model and Provider Setup

- Added dynamic model catalog lookup through `models.dev`.
- Added Codex OAuth import options.
- Added Ollama provider support.
- Added OpenAI-compatible Responses API support.
- Added provider-aware reasoning options, reasoning budget configuration, and reserved output token settings.
- Added vision capability detection and image-stripping fallback when the configured model does not support vision.
- Added an efficient-model path for routine background operations such as generated session titles.

## Telegram

- Added Telegram session control commands for listing, creating, attaching, and deleting sessions.
- Render Telegram replies as Markdown.
- Persist Telegram update offsets before restart commands.
- Improved Telegram approval, dashboard status, session routing, and delivery handling.

## Desktop and Local Operation

- Added daemon tray support on Windows and desktop Linux/macOS environments.
- Added daemon restart controls and more reliable daemon shutdown behavior.
- Added browser runtime setup support.
- Added a config switch to disable sandboxing when needed for local workflows.
- Source builds now build and embed the WebUI; install Bun before using a source-build path such as `cargo install`.

## Fixes and Reliability

- Fixed WebUI session switching, pending-input submission, activity folding, markdown rendering, composer styling, and image attachment handling.
- Fixed browser and daemon request hangs.
- Fixed terminal unread-output and short-yield output capture behavior.
- Fixed runtime compaction and workflow-binding recovery loops.
- Fixed daemon startup, restart, shutdown, and session cleanup edge cases.
- Fixed Windows session startup issues and background child-process console behavior.
- Fixed Telegram approval flow and session command routing.
- Fixed model/provider compatibility edge cases for Codex OAuth, OpenAI-compatible providers, reasoning options, and vision handling.

## Compatibility Notes

- Project-scoped code sessions use the project directory as their workspace.
- Workspace apps remain under `~/daat-locus-workspace/apps`.
- If you build from source, Bun must be installed because the WebUI is embedded during the Rust build.
